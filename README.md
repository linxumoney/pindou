# 拼豆图片转换器

一个纯前端、可静态部署的拼豆图片转换网站。用户上传图片后，网站会在浏览器本地完成图片压缩、像素化、颜色量化、拼豆色板匹配、图纸预览、用色统计和高清 PNG 导出。

## 功能

- 支持 JPG、PNG、WEBP、GIF、BMP 等常见图片上传
- 支持点击上传和拖拽上传
- 上传后显示原图预览
- 大图自动降采样到合理尺寸，避免处理卡顿
- 支持按格数或按成品尺寸生成图纸
- 支持 2mm / 5mm 拼豆规格
- 支持锁定原图宽高比例
- 支持裁切填满或完整留白
- 支持透明、白色、自定义背景
- 支持原图近似色模式和拼豆色板匹配模式
- 内置常用拼豆色板，颜色包含编号、名称和色值
- 支持限制最大颜色数量
- 支持显示/隐藏网格线
- 支持显示/隐藏颜色编号
- 支持缩放预览和滚动查看大图纸
- 自动统计每种颜色的预计颗数
- 支持导出高清 PNG 图纸，适合保存或打印

## 本地运行

这是一个无后端、无构建依赖的静态项目。

直接用浏览器打开：

```text
bead-studio/index.html
```

也可以用任意静态服务器运行，例如 VS Code Live Server、Nginx、GitHub Pages、Vercel、Netlify。

## 部署

将 `bead-studio` 目录作为静态网站目录部署即可。

主要文件：

```text
bead-studio/
  index.html   页面结构
  styles.css   页面样式和响应式布局
  app.js       图片处理、拼豆算法、Canvas 渲染、PNG 导出
  README.md    项目说明
```

## 使用方式

1. 点击或拖拽上传图片。
2. 选择按格数或按成品尺寸生成图纸。
3. 根据实际拼豆选择 2mm 或 5mm。
4. 调整色彩数量、图片适配方式、背景和色彩模式。
5. 查看图纸预览和用色清单。
6. 需要打印时开启导出编号，然后点击“下载 PNG”。

## 浏览器兼容性

支持现代浏览器：

- Chrome
- Edge
- Safari
- Firefox

图片转换全部在浏览器本地完成，不会上传到服务器。

## 已知限制

- 超过 `200 × 200` 的拼豆图纸会被限制到最大 200 格，避免浏览器长时间卡顿。
- 内置色板是常用拼豆色的近似集合，不对应某一个品牌的完整官方色号。
- GIF 会按浏览器解码结果处理首帧或默认帧，不生成动画图纸。

### Core Features

#### 1. 图片上传

实现图片上传功能，要求：

- 支持 JPG、PNG、WEBP 等常见图片格式
- 支持点击上传
- 支持拖拽上传
- 上传后立即显示原图预览
- 对超大图片进行合理压缩或降采样，避免页面卡死
- 上传失败或格式不支持时，给出清晰提示

#### 2. 智能像素化

实现图片像素化处理，要求：

- 根据用户设置生成拼豆点阵
- 支持设置点阵宽度和高度
- 支持锁定宽高比例
- 支持常见拼豆规格：
  - 2mm 拼豆
  - 5mm 拼豆
- 支持选择成品尺寸或格数
- 像素块边界清晰，不模糊
- 算法应尽量保留原图主体轮廓和主要颜色

#### 3. 色彩还原与拼豆色匹配

实现拼豆颜色匹配逻辑，要求：

- 内置一套常用拼豆色板
- 将图片颜色自动匹配到最接近的拼豆颜色
- 支持显示使用到的颜色列表
- 每种颜色显示：
  - 色块
  - 颜色名称
  - 颜色编号或标识
  - 预计使用颗数
- 支持用户切换：
  - 原图近似色模式
  - 拼豆色板匹配模式
- 色彩匹配应尽量稳定、自然，不出现明显异常偏色

#### 4. 拼豆图纸预览

生成适合手工制作的拼豆图纸，要求：

- 使用网格展示点阵
- 每个格子代表一颗拼豆
- 支持显示或隐藏网格线
- 支持显示或隐藏颜色编号
- 支持缩放预览
- 支持移动查看大图纸
- 图纸区域清晰，不拥挤
- 大尺寸图纸也应尽量保持流畅

#### 5. 参数调节

提供实用但不冗余的参数控制，至少包括：

- 点阵宽度
- 点阵高度
- 是否锁定比例
- 拼豆尺寸：2mm / 5mm
- 色彩数量限制
- 是否启用拼豆色板匹配
- 网格线开关
- 编号显示开关
- 背景色选择，支持透明 / 白色 / 自定义颜色

用户调整参数后，应能实时或点击按钮重新生成图纸。

#### 6. 图纸导出

实现 PNG 高清导出功能，要求：

- 可导出当前拼豆图纸为 PNG
- 导出图纸应包含清晰网格
- 可选择是否包含颜色编号
- 导出图片清晰，适合打印或保存
- 文件名应有意义，例如 `bead-pattern.png`
- 导出过程不能破坏当前页面状态

### Algorithm Requirements

图片转拼豆逻辑应至少包含：

1. 读取上传图片
2. 根据目标点阵尺寸缩放图片
3. 对缩放后的像素进行采样
4. 将每个像素块转换为代表色
5. 根据色板进行最近色匹配
6. 生成二维点阵数据
7. 统计每种颜色使用数量
8. 使用 Canvas 或 DOM 渲染图纸
9. 支持导出高清 PNG

色彩匹配建议：

- 使用 RGB 距离或更合理的感知色差算法
- 可加入简单的亮度、饱和度修正
- 避免颜色数量过多导致实际制作困难
